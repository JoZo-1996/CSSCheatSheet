🤔 Was ist CSS überhaupt?

💡
Einfache Erklärung: HTML ist das Skelett einer Webseite (der Inhalt). CSS ist die Kleidung — es bestimmt, wie alles aussieht: Farben, Schriftarten, Abstände, Größen und mehr. Ohne CSS wäre jede Webseite nur schwarzer Text auf weißem Hintergrund.

🏗️ HTML = Inhalt

HTML erstellt die Elemente: Überschriften, Absätze, Bilder, Buttons, Links.

<!-- HTML: Was da ist -->
<h1>Hallo Welt</h1>
<p>Ein Absatz</p>
<button>Klick mich</button>

🎨 CSS = Aussehen

CSS gestaltet die Elemente: Farbe, Größe, Abstand, Rahmen, Schrift.

/* CSS: Wie es aussieht */
h1 {
  color: blau;
  font-size: 32px;
}
button {
  background: green;
}

📁 Wie bindet man CSS ein?

Am besten als eigene .css Datei — das ist die empfohlene Methode für alle Projekte.

<!-- Im <head> deiner HTML-Datei -->
<link rel="stylesheet"
      href="style.css">

<!-- Oder direkt im <head> -->
<style>
  p { color: red; }
</style>
01

Grundaufbau einer CSS-Regel

Der Bauplan
💡
Jede CSS-Regel hat immer dasselbe Muster: Wen willst du stylen? (Selektor) → Was soll sich ändern? (Eigenschaft) → Wie soll es aussehen? (Wert)

🏗️ Der Grundaufbau

Stell dir vor: Du willst alle Überschriften blau machen.

h1 {
  color: blue;
  font-size: 32px;
  /* Das hier ist ein Kommentar */
  /* Browser ignoriert Kommentare */
}

/* Aufbau:
   Selektor { Eigenschaft: Wert; } */

🏆 Welches CSS "gewinnt"?

Wenn mehrere Regeln dasselbe Element ansprechen, gelten diese Regeln (von wichtigsten zu unwichtigsten):

1. Inline-Stil am HTML-Tag selbst (gewinnt fast immer)

2. ID-Selektoren (#mein-element)

3. Klassen-Selektoren (.meine-klasse)

4. Element-Selektoren (p, h1, div)

color: red !important;
/* !important überschreibt ALLES */
/* Nur im Notfall verwenden! */

📌 Kaskade — was bedeutet das?

CSS steht für Cascading Style Sheets. "Cascading" = "kaskadierend" = wie ein Wasserfall, der von oben nach unten fließt.

Regeln die weiter unten in der CSS-Datei stehen, überschreiben Regeln die weiter oben stehen (wenn sie gleich stark sind).

p { color: red; }
p { color: blue; }
/* p wird BLAU — die 2. Regel gewinnt */
02

Selektoren

„Wen will ich stylen?"
💡
Ein Selektor ist wie ein Finger der auf etwas zeigt. Du sagst CSS: „Mach dieses Element schön!" Es gibt viele Wege, ein Element auszuwählen.
SelektorBeispielWas er macht (einfach erklärt)
Elementp { }Alle Absätze (<p>) auf der gesamten Seite
.klasse.btn { }Alle Elemente mit class="btn" — mehrere Elemente möglich!
#id#header { }Das eine Element mit id="header" — jede ID darf nur einmal vorkommen!
** { }Alle Elemente auf der Seite (der „Alles-Selektor")
A, Bh1, h2 { }Mehrere auf einmal — beide h1 UND h2 bekommen denselben Stil
A Bdiv p { }Alle <p>-Elemente die irgendwo innerhalb eines <div> sind
A > Bul > li { }Nur <li> die direkte Kinder des <ul> sind — keine tiefer liegenden
A + Bh1 + p { }Das <p> das direkt nach einem <h1> kommt (Nachbar)
[attr="wert"][type="text"] { }Alle Elemente mit einem bestimmten Attribut und Wert

🔗 Selektoren kombinieren

Du kannst Selektoren miteinander verbinden, um genauer zu zielen:

/* Element UND Klasse */
p.intro { color: blue; }
/* Nur <p>-Elemente mit class="intro" */

/* Klasse innerhalb von anderem Element */
.nav .link { font-weight: 700; }
/* .link nur wenn es in .nav ist */

/* Eingabefeld beim Anklicken */
input:focus {
  border-color: blue;
}

🆚 Klasse vs. ID — was ist der Unterschied?

Klasse (.name): Kann mehrfach auf derselben Seite verwendet werden. Ideal für Dinge die mehrfach vorkommen (Buttons, Karten, Texte).

ID (#name): Darf nur EINMAL pro Seite vorkommen. Gut für eindeutige Bereiche wie die Navigationsleiste oder den Footer.

<!-- Klasse: mehrfach erlaubt -->
<button class="btn">OK</button>
<button class="btn">Abbruch</button>

<!-- ID: nur einmal! -->
<nav id="hauptmenu">...</nav>

Vorrang-Regeln (Spezifität)

Welche Regel setzt sich durch?
💡
Stell dir vor, zwei CSS-Regeln wollen dasselbe Element anders einfärben. Welche gewinnt? Das bestimmt die Spezifität — so etwas wie eine Punkte-Bewertung. Wer mehr Punkte hat, gewinnt.

🏆 Die Rangfolge (höchste zuerst)

style="" direkt am HTML-Tag → stärkste
#id-Selektor → sehr stark
.klasse und [attribut] → mittel
element (p, h1, div) → schwächste

📊 Punkte-Beispiele

SelektorPunkte
p1 Pkt.
.box10 Pkt.
p.box11 Pkt.
#nav100 Pkt.
style=""1000 Pkt.
!importantgewinnt immer
⚠️
!important nur im Notfall verwenden — es macht den Code schwer zu pflegen.
04

Maßeinheiten

Wie groß ist etwas?
💡
In CSS gibt es verschiedene Einheiten — wie beim Messen in echt (Meter, Zentimeter, Prozent...). Manche sind fest (immer gleich groß), andere sind flexibel (passen sich an den Bildschirm an).
EinheitBeispielEinfach erklärt
📏 Feste Einheiten (immer gleich groß)
px16pxPixel — die häufigste Einheit. 1px ist ein einzelner Bildschirmpunkt. Gut für Rahmen, kleine Details.
📐 Flexible Einheiten (passen sich an)
%50%Prozent des übergeordneten Elements. Wenn das Elternelement 400px breit ist, sind 50% = 200px.
em1.5emVerhältnis zur Schriftgröße des Elternelements. Vorsicht: Kann sich bei Schachtelung summieren!
rem1remWie em, aber immer relativ zur Grundschriftgröße der Seite (meistens 16px). Empfehlung für Anfänger!
📱 Bildschirm-Einheiten (relativ zum Browser-Fenster)
vw100vw1vw = 1% der Browserbreite. 100vw = volle Browserbreite. Gut für große Banner.
vh100vh1vh = 1% der Browserhöhe. 100vh = volle Browserhöhe. Gut für Vollbild-Bereiche.
🔲 Nur für CSS Grid
fr1fr„Fraction" (Anteil) — teilt den freien Platz auf. 1fr 2fr = ein Drittel + zwei Drittel.

🤔 px, em oder rem — was nehmen?

px → für feste Größen: Rahmen, Schatten, kleine Icons

rem → für Schriftgrößen und Abstände (passt sich an Nutzer-Einstellungen an)

% → für Breiten von Containern (z.B. width: 100%)

vw/vh → für Elemente die den ganzen Bildschirm füllen sollen

⏱️ Zeit-Einheiten (für Animationen)

Für Übergänge und Animationen braucht man Zeit-Einheiten:

transition-duration: 0.3s;
/* 0.3 Sekunden */

animation-duration: 1500ms;
/* 1500 Millisekunden = 1.5 Sekunden */
05

Das Box-Modell

Wie viel Platz nimmt ein Element ein?
💡
Jedes HTML-Element ist eine Box. Diese Box hat 4 Schichten: Inhalt → Innenabstand → Rahmen → Außenabstand. Stell dir eine Postkarte im Umschlag in einem Paket vor!

📦 Die 4 Schichten

🟡 margin (Außenabstand)
🔴 border (Rahmen)
🟢 padding (Innenabstand)
🔵 content (Inhalt)

📏 Margin, Padding, Border

margin = Abstand nach außen (zu anderen Elementen)

padding = Abstand nach innen (zwischen Rahmen und Inhalt)

border = der Rahmen selbst

/* Alle 4 Seiten gleich */
margin: 10px;

/* Oben/Unten  Links/Rechts */
padding: 10px 20px;

/* Oben Rechts Unten Links */
margin: 10px 20px 30px 40px;
/* Merkregel: Uhrzeigersinn! */

/* Nur eine Seite */
margin-top: 20px;
padding-left: 15px;

⚠️ Wichtiger Tipp: box-sizing

Standardmäßig kommt Padding und Border noch drauf zur angegebenen Breite — das ist verwirrend!

Mit border-box ist alles intuitiver: die Breite schließt Padding und Border bereits ein.

/* Das solltest du IMMER am Anfang
   deiner CSS-Datei einfügen! */
*, *::before, *::after {
  box-sizing: border-box;
}

/* Rahmen */
border: 2px solid #333;
border-radius: 8px; /* runde Ecken */
border-radius: 50%; /* Kreis! */

Text & Schrift

Wie Text aussieht

🔤 Schriftart & Größe

/* Schriftart (immer Fallback angeben) */
font-family: 'Roboto', Arial, sans-serif;

/* Größe */
font-size: 1rem; /* = 16px Standard */
font-size: 24px;

/* Dicke (Fettschrift) */
font-weight: 400; /* normal */
font-weight: 700; /* fett */
font-weight: bold;

/* Kursiv */
font-style: italic;

/* Zeilenhöhe */
line-height: 1.6; /* angenehm zum Lesen */

📝 Text formatieren

/* Textfarbe */
color: #333;

/* Ausrichtung */
text-align: left;     /* links */
text-align: center;   /* mittig */
text-align: right;    /* rechts */

/* Unterstrichen / Durchgestrichen */
text-decoration: underline;
text-decoration: none;  /* entfernt Unterstrich */

/* Großschreibung */
text-transform: uppercase; /* ALLES GROß */
text-transform: capitalize; /* Jeden Ersten Buchstaben */

/* Abstand zwischen Buchstaben */
letter-spacing: 0.1em;

/* Text-Schatten */
text-shadow: 2px 2px 4px rgba(0,0,0,0.3);

🌐 Google Fonts nutzen

Du kannst kostenlos schöne Schriften von Google einbinden:

<!-- 1. Im HTML <head> einfügen -->
<link
  href="https://fonts.googleapis.com/
  css2?family=Roboto&display=swap"
  rel="stylesheet">

/* 2. In CSS verwenden */
body {
  font-family: 'Roboto', sans-serif;
}

💡 HSL — Farbe einfach wählen

HSL steht für Hue (Farbton 0-360°), Saturation (Sättigung 0-100%), Lightness (Helligkeit 0-100%).

Das macht es leichter, Farben zu variieren:

/* Rot */
color: hsl(0, 80%, 50%);

/* Helleres Rot - nur Helligkeit ändern */
color: hsl(0, 80%, 70%);

/* Transparentes Rot */
color: hsla(0, 80%, 50%, 0.5);
08

Anzeige & Position

Wie und wo erscheint ein Element?

📺 display-Eigenschaft

block = nimmt volle Breite ein, neue Zeile (div, p, h1)

inline = nur so breit wie der Inhalt, bleibt in der Zeile (span, a)

inline-block = wie inline, aber Breite/Höhe einstellbar

none = Element ist vollständig unsichtbar und nimmt keinen Platz ein

display: block;
display: inline;
display: inline-block;
display: none;  /* verstecken */
display: flex;  /* → Flexbox-Kapitel */
display: grid;  /* → Grid-Kapitel */

📍 position-Eigenschaft

static = Normalfluss (Standard)

relative = verschoben relativ zu seiner normalen Position

absolute = aus dem Fluss heraus, positioniert im nächsten positioned-Elternelement

fixed = bleibt beim Scrollen auf dem Bildschirm fixiert

sticky = normal, aber bleibt oben „kleben" wenn man scrollt

position: fixed;
top: 0;
left: 0;
/* z.B. für fixe Navigationsleiste */

🗂️ z-index — wer liegt oben?

Wenn Elemente übereinanderliegen, bestimmt z-index wer sichtbar ist. Höherer Wert = weiter vorne.

.popup {
  position: fixed;
  z-index: 1000; /* liegt über allem */
}

.header {
  position: sticky;
  top: 0;
  z-index: 100;
}

Flexbox

Elemente in einer Reihe oder Spalte anordnen
💡
Flexbox ist wie eine flexible Schachtel. Du kannst bestimmen, wie die Inhalt-Boxen darin angeordnet werden: nebeneinander, untereinander, wie viel Abstand dazwischen, ob sie sich ausdehnen usw.

👁️ Live-Vorschau

Box 1
Box 2
Box 3

Diese 3 Boxen liegen nebeneinander, weil ihr Container display: flex hat.

🐣 Flexbox starten (am Container)

.container {
  display: flex;  /* Flexbox aktivieren */

  /* Richtung: nebeneinander oder untereinander */
  flex-direction: row;     /* → nebeneinander */
  flex-direction: column;  /* ↓ untereinander */

  /* Abstand zwischen den Elementen */
  gap: 1rem;

  /* Umbrechen wenn kein Platz mehr */
  flex-wrap: wrap;
}

↔️ Ausrichtung horizontal (justify-content)

/* Alle nach links (Standard) */
justify-content: flex-start;

/* Alle nach rechts */
justify-content: flex-end;

/* Alle in der Mitte */
justify-content: center;

/* Gleichmäßig verteilt */
justify-content: space-between;

/* Mit Abstand außen */
justify-content: space-around;

↕️ Ausrichtung vertikal (align-items)

/* Am oberen Rand */
align-items: flex-start;

/* Am unteren Rand */
align-items: flex-end;

/* In der Mitte — sehr nützlich! */
align-items: center;

/* Strecken (Standard) */
align-items: stretch;

⭐ Perfekt zentrieren

Das häufigste Problem für Anfänger: etwas zentrieren. Mit Flexbox ganz einfach:

.container {
  display: flex;
  justify-content: center; /* horizontal */
  align-items: center;    /* vertikal */
  height: 100vh;
}
/* → Inhalt ist perfekt zentriert! */

📦 Kinder-Elemente steuern

.kind-element {
  /* Wie viel darf es wachsen? */
  flex-grow: 1;  /* füllt freien Platz */
  flex-grow: 0;  /* wächst nicht */

  /* Mindestbreite */
  flex-basis: 200px;

  /* Kurzschrift: wachsen schrumpfen basis */
  flex: 1 1 200px;

  /* Sich selbst anders ausrichten */
  align-self: flex-end;
}

CSS Grid

Elemente in einem Raster anordnen
💡
Grid ist wie ein Tabellen-Layout — aber viel mächtiger! Du definierst Spalten und Zeilen, und die Elemente ordnen sich automatisch ein. Perfekt für komplexe Seiten-Layouts.

👁️ Live-Vorschau: 3 Spalten (1fr 2fr 1fr)

A
B (nimmt 2 Spalten ein)
C
D
E

🏁 Grid starten

.container {
  display: grid;

  /* 3 Spalten: 1/4, 1/2, 1/4 */
  grid-template-columns: 1fr 2fr 1fr;

  /* 3 gleich breite Spalten */
  grid-template-columns: repeat(3, 1fr);

  /* Automatisch so viele wie reinpassen */
  grid-template-columns:
    repeat(auto-fill, minmax(250px, 1fr));

  /* Abstand zwischen Zellen */
  gap: 1rem;
}

🗺️ Element über mehrere Zellen strecken

.element {
  /* Über 2 Spalten strecken */
  grid-column: span 2;

  /* Über 3 Zeilen strecken */
  grid-row: span 3;

  /* Exakte Position: Spalte 2 bis 4 */
  grid-column: 2 / 4;

  /* Die ganze Breite einnehmen */
  grid-column: 1 / -1;
}

🏗️ Benannte Bereiche (Template Areas)

Du kannst Bereiche mit Namen versehen — sehr übersichtlich!

.container {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

CSS Variablen

Werte einmal definieren, überall verwenden
💡
Stell dir vor, du hast auf 50 Seiten die gleiche blaue Farbe verwendet. Jetzt soll sie grün werden. Ohne Variablen: 50 Stellen ändern. Mit Variablen: eine Stelle ändern, alles aktualisiert sich automatisch!

📦 Variablen definieren & nutzen

/* Variablen immer in :root definieren */
/* (damit sie überall verfügbar sind) */
:root {
  --hauptfarbe: #3b5bdb;
  --text-farbe: #333;
  --abstand: 1rem;
  --rahmen-radius: 8px;
}

/* Verwenden mit var() */
button {
  background: var(--hauptfarbe);
  padding: var(--abstand);
  border-radius: var(--rahmen-radius);
}

🌙 Dark Mode mit Variablen

Mit Variablen kannst du einen Dark Mode ganz einfach umsetzen:

:root {
  --bg: white;
  --text: #333;
}

/* Wenn Nutzer Dark Mode bevorzugt */
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #1a1a1a;
    --text: #eeeeee;
  }
}

body {
  background: var(--bg);
  color: var(--text);
}

Transformationen

Elemente drehen, schieben, vergrößern
💡
Mit transform kannst du ein Element visuell verschieben, drehen oder vergrößern — ohne das Layout der anderen Elemente zu stören!

👁️ Beispiele

normal

normal

rotate

rotate(20deg)

scale

scale(1.3)

translate

translateX(15px)

skew

skewX(15deg)

🔧 Alle Transform-Funktionen

/* Verschieben */
transform: translateX(20px);   /* nach rechts */
transform: translateY(-10px);  /* nach oben */
transform: translate(20px, -10px);

/* Drehen */
transform: rotate(45deg);    /* 45 Grad */
transform: rotate(-90deg);   /* gegen Uhrzeigersinn */

/* Vergrößern/Verkleinern */
transform: scale(1.5);   /* 1.5x größer */
transform: scale(0.5);   /* halb so groß */

/* Kombinieren */
transform: rotate(45deg) scale(1.2);

🎯 Drehpunkt ändern (transform-origin)

Standardmäßig dreht/skaliert sich alles von der Mitte. Das kannst du ändern:

/* Drehpunkt oben links */
transform-origin: top left;

/* Drehpunkt unten rechts */
transform-origin: bottom right;

/* Mit Koordinaten */
transform-origin: 0% 50%;

Übergänge (Transitions)

Sanfte Veränderungen beim Hover, Klick & Co.
💡
Ohne Transition: Farbe wechselt sofort abrupt. Mit Transition: Farbe wechselt sanft über eine bestimmte Zeit. Das macht Webseiten viel professioneller!

👁️ Demo — fahre mit der Maus über den Button!

⚙️ Transition einrichten

.button {
  background: blue;
  color: white;

  /* Alle Änderungen sanft über 0.3 Sek. */
  transition: all 0.3s ease;
}

.button:hover {
  background: darkblue;
  transform: scale(1.05);
  /* → sanfter Übergang dank transition! */
}

⏱️ Geschwindigkeit & Timing

/* ease = langsam, schnell, langsam (Standard) */
transition-timing-function: ease;

/* linear = gleichmäßig */
transition-timing-function: linear;

/* ease-in = startet langsam */
transition-timing-function: ease-in;

/* ease-out = endet langsam */
transition-timing-function: ease-out;

/* Kurzschrift: Eigenschaft Dauer Timing */
transition: background 0.3s ease,
            transform 0.2s ease-out;
14

Animationen

Bewegte Elemente ohne JavaScript
💡
Transitions laufen einmal (beim Hover z.B.). Animationen laufen automatisch und können sich wiederholen. Du definierst Schlüsselpunkte (@keyframes) und CSS macht den Rest!

👁️ Demo — diese Box pulsiert automatisch

💙

🎬 Animation erstellen

/* 1. Keyframes definieren (der Ablauf) */
@keyframes pulsieren {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.1); }
  100% { transform: scale(1); }
}

/* 2. Animation am Element nutzen */
.mein-element {
  animation-name: pulsieren;
  animation-duration: 2s;        /* Dauer */
  animation-iteration-count: infinite; /* Wiederholen */
}

⚙️ Weitere Animation-Optionen

.element {
  /* Kurzschrift */
  animation: pulsieren 2s ease-in-out infinite;

  /* Richtung: vorwärts-rückwärts-vorwärts... */
  animation-direction: alternate;

  /* 1 Sekunde warten bevor Start */
  animation-delay: 1s;

  /* Am Ende bleibt Element im letzten Zustand */
  animation-fill-mode: forwards;

  /* Animation pausieren */
  animation-play-state: paused;
}

🌟 Einblend-Animation (Beispiel)

@keyframes einblenden {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.karte {
  animation: einblenden 0.5s ease-out;
}
/* Element fährt beim Laden sanft von unten ein */

Pseudo-Klassen & Pseudo-Elemente

Spezielle Zustände und unsichtbare Elemente
💡
Pseudo-Klassen (mit einem Doppelpunkt) beschreiben einen Zustand: „Wenn die Maus drüber ist", „Wenn es das erste Kind ist", „Wenn der Link besucht wurde".
Pseudo-Elemente (mit zwei Doppelpunkten) erzeugen virtuelle Elemente: vor oder nach dem Inhalt.

🖱️ Interaktions-Zustände

/* Wenn Maus drüber fährt */
a:hover { color: blue; }

/* Wenn angeklickt wird */
button:active { transform: scale(0.95); }

/* Wenn Eingabefeld aktiv ist */
input:focus { border-color: blue; }

/* Checkbox die angehakt ist */
input:checked { accent-color: green; }

/* Besuchter Link */
a:visited { color: purple; }

🔢 Positions-Pseudo-Klassen

/* Erstes Kind-Element */
li:first-child { font-weight: bold; }

/* Letztes Kind-Element */
li:last-child { border: none; }

/* Jedes 2. Element */
li:nth-child(2n) { background: #f0f0f0; }

/* Jedes gerade Element */
tr:nth-child(even) { background: #f5f5f5; }

/* Wenn ein Element keine Kinder hat */
p:empty { display: none; }

✨ Pseudo-Elemente (::)

/* Inhalt VOR dem Element einfügen */
.preis::before {
  content: "€ ";
}

/* Inhalt NACH dem Element einfügen */
.tipp::after {
  content: " ★";
  color: gold;
}

/* Ersten Buchstaben stylen */
p::first-letter {
  font-size: 2em;
  float: left;
}

/* Platzhalter in Eingabefeldern */
input::placeholder { color: #aaa; }

Media Queries

Responsives Design für alle Bildschirmgrößen
💡
Media Queries sind wie Fragen an den Browser: „Ist der Bildschirm kleiner als 768px?" Je nach Antwort werden andere CSS-Regeln angewendet. So sieht deine Seite auf dem Handy genauso gut aus wie auf dem großen Monitor!

📱 Grundstruktur

/* Grundstile für Mobilgeräte */
.container {
  width: 100%;
  padding: 1rem;
}

/* Ab 768px → Tablet */
@media (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Ab 1024px → Desktop */
@media (min-width: 1024px) {
  .container {
    width: 980px;
  }
}

🎨 Weitere Media-Bedingungen

/* Automatisch Dark Mode aktivieren */
@media (prefers-color-scheme: dark) {
  body { background: #111; }
}

/* Quer- oder Hochformat */
@media (orientation: landscape) {
  /* Querformat */
}

/* Drucken */
@media print {
  .navigation { display: none; }
}

/* Nutzer mag keine Animationen */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; }
}

Nützliche CSS-Funktionen

Berechnungen und clevere Werte

🧮 calc() — Rechnen in CSS

Du kannst direkt in CSS rechnen — sogar mit gemischten Einheiten!

/* Volle Breite minus Seitenleiste */
width: calc(100% - 280px);

/* Schriftgröße die mit Viewport wächst */
font-size: calc(1rem + 0.5vw);

/* Variable mit Berechnung */
margin-top: calc(var(--abstand) * 2);

📏 min(), max(), clamp()

Diese Funktionen helfen bei responsivem Design ohne Media Queries:

/* Maximal 600px breit, aber nie mehr als 100% */
width: min(600px, 100%);

/* Mindestens 2rem, nie kleiner */
padding: max(2rem, 5%);

/* clamp(min, ideal, max) */
/* Schrift: min 14px, ideal 2vw, max 22px */
font-size: clamp(14px, 2vw, 22px);

/* Abstand: min 1rem, ideal 5vw, max 3rem */
padding: clamp(1rem, 5vw, 3rem);

🎨 Farb-Funktionen

/* RGB mit Durchsichtigkeit */
color: rgb(255 0 0 / 50%); /* 50% transparent */

/* HSL — einfache Farbanpassung */
color: hsl(220, 80%, 50%);

/* Linearer Farbverlauf */
background: linear-gradient(to right, #f00, #00f);

/* Radialer Farbverlauf (von Mitte nach außen) */
background: radial-gradient(circle, #fff, #000);

🔧 Filter-Effekte

Mit filter kannst du visuelle Effekte auf Elemente anwenden:

/* Unschärfe */
filter: blur(4px);

/* Helligkeit (0=schwarz, 2=doppelt so hell) */
filter: brightness(0.8);

/* Graustufen */
filter: grayscale(100%);

/* Kombination */
filter: brightness(0.9) contrast(1.1);

/* Schatten (auch auf transparente Bilder!) */
filter: drop-shadow(2px 4px 6px black);
